# Сервис транскрибации и анализа перевода (FastAPI)

Веб‑приложение на FastAPI для:
- транскрибации аудио через OpenAI Whisper;
- экспертного анализа устного перевода студента по заданному исходному тексту с использованием GPT (настраиваемый системный промпт);
- простых HTML‑страниц для загрузки файлов и просмотра результата.

Маршруты сгруппированы под префиксом `/api/v1`.

## Возможности
- Транскрибация аудио в текст (язык задаётся параметром формы `language`).
- Анализ аудио‑перевода против исходного текста с бальной системой штрафов (редактируемый промпт).
- Веб‑формы для быстрой проверки: загрузка файла, выбор языка, просмотр результата.
- REST‑эндпоинт для машинной интеграции (JSON‑ответ).

## Технологии
- Python 3.10
- FastAPI + Uvicorn
- Jinja2 шаблоны (HTML)
- OpenAI API (`whisper-1` для STT, GPT‑модели для анализа)

## Быстрый старт

### 1) Установите зависимости
Рекомендуемое окружение: Windows/WSL/macOS/Linux с Python 3.10.

```bash
# Создать и активировать виртуальное окружение (Windows PowerShell)
python -m venv .venv
. .venv/Scripts/Activate.ps1

# Установить зависимости
pip install -r requirements.txt
```

Если `pip` ругается на кодировку `requirements.txt`, сохраните файл в UTF‑8 (сейчас он может быть в UTF‑16), либо выполните по одному:

```bash
pip install fastapi==0.135.1 uvicorn==0.41.0 jinja2==3.1.6 openai==2.24.0 python-multipart==0.0.22
```

### 2) Настройте переменные окружения
Приложению нужен ключ OpenAI. Рекомендуется хранить его в переменной окружения:

```bash
# Windows PowerShell
$Env:OPENAI_API_KEY = "sk-..."
```

> Безопасность: не храните ключ в коде или VCS. Текущая версия кода использует создание клиента OpenAI напрямую; рекомендуется перейти на чтение ключа из переменных окружения.

### 3) Запуск сервера
Режим разработки (автоперезапуск):

```bash
uvicorn main:app --reload --host 127.0.0.1 --port 8080
```

Продакшен‑команда согласно `amvera.yml`:

```bash
uvicorn main:app --host 0.0.0.0 --port 80
```

После запуска:
- UI формы: http://127.0.0.1:8080/api/v1/transcribe и http://127.0.0.1:8080/api/v1/analyze
- OpenAPI/Swagger: http://127.0.0.1:8080/docs
 - Новые формы: http://127.0.0.1:8080/api/v1/ask и http://127.0.0.1:8080/api/v1/transcribe_model

## Эндпоинты

### HTML‑страницы
- `GET /api/v1/transcribe` — форма загрузки аудио и выбора языка (`ru`, `en`, `de`, `fr` — предустановлено в шаблоне, можно расширять).
- `POST /api/v1/transcribe` — обрабатывает загрузку, возвращает результат транскрибации на странице.
- `GET /api/v1/analyze` — форма с полями `system_prompt`, `user_prompt` и `gpt_version` (модель GPT, по умолчанию в UI — `gpt-audio-mini-2025-12-15`).
- `POST /api/v1/analyze` — принимает аудио, `system_prompt`, `user_prompt`, `gpt_version`; анализирует и показывает результат.
 - `GET /api/v1/ask` — форма для текстового запроса к модели (system + user prompt).
 - `POST /api/v1/ask` — отправляет system/user промпты выбранной модели и показывает ответ.
 - `GET /api/v1/transcribe_model` — форма транскрибации с выбором модели (например, `whisper-1`).
 - `POST /api/v1/transcribe_model` — принимает аудио и имя модели транскрибации, возвращает текст на странице.

### REST API
- `POST /api/v1/spech_to_text` — транскрибация аудио. Возвращает JSON `{ "text": "..." }`.

Параметры запроса:
- Form‑данные: `file` — аудиофайл (обязателен).

Пример `curl`:

```bash
curl -X POST "http://127.0.0.1:8080/api/v1/spech_to_text" \
  -H "Accept: application/json" \
  -F "file=@sample.wav"
```

## Поддерживаемые форматы и ограничения
- Форматы: `mp3`, `mp4`, `mpeg`, `mpga`, `m4a`, `wav`, `webm`
- Максимальный размер файла: 25 МБ (лимит API)

## Настройки и поведение
- Ограничение размеров: файлы больше 25 МБ отклоняются.
- Поддерживаемые расширения проверяются до вызова API.
- В `analyze` системный промпт предзаполнен строгой академической рубрикой (B2–C1, научные конференции) и может изменяться в форме.
- Параметр `gpt_version` передаётся как имя модели для анализа (пример в UI — `gpt-audio-mini-2025-12-15`).

## Замечания по безопасности
- Храните API‑ключи только в переменных окружения/секретах. Не коммитьте их в репозиторий.
- Рекомендуется заменить явную передачу `api_key` в коде на чтение из переменных окружения (например, `OpenAI()` без параметров при наличии `OPENAI_API_KEY`).
